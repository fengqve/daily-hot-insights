  • microsoft/agent-framework | Python / 微软 AI Agent 框架 | 构建、编排和部署多 Agent 工作流的 Python 框架，同时支持 .NET | **洞察**：微软继续押注 Agent 基础设施，.NET 兼容性暗示企业级 .NET 生态的存量价值，不甘只做 OpenAI 的调用方。
  • block/goose | Rust / 可扩展 AI Agent | 开源可扩展 AI Agent，支持安装、执行、编辑、测试，可对接任意 LLM | **洞察**：Rust 运行时带来性能优势，支持任意 LLM 意味着去供应商锁定，灵活性对标 Claude Code 但完全开源。
  • Mastra | TypeScript / Agent 框架 | TypeScript 优先的 Agent 开发框架，v1.0 于 2026年1月发布，支持 RAG、工作流、工具集成和可观测内存 | **洞察**：TypeScript 生态的 Agent 框架空白终于被填补，1.0 发布节奏说明生产可用性已成熟，对前端/全栈团队友好。
  • Dify | 开源 / 低代码 AI Agent 平台 | 低代码可视化 AI Agent 创建平台，支持数百种 LLM、RAG、Function Calling 和 ReAct 策略 | **洞察**：Dify 已成为 AI 应用层的"WordPress"，降低了非工程师创建 Agent 的门槛，社区生态持续扩大。
  • **洞察**：本周 GitHub 主旋律是"AI Agent 架构透明化"——OpenClaw 25万星破纪录、Claude Code 源码泄露催生 clawe-code、Mastra 1.0 发布标志着 TypeScript Agent 框架进入生产阶段。Rust 运行时（goose）和低代码可视化（Dify）代表两条并行路线：性能极限 vs 普及化。
